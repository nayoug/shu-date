# 前端重设计交接包

这个目录最初用于存放 `shu-date` 独立前端重设计方案的交接材料，方便在**不直接改动现有 Express + EJS 前端**的前提下进行评审、讨论与后续迁移。

当前这条分支已经不只是静态交接包，还额外落了一版**可运行的新版共存预览**：
- 新增 `views/v2/` 与 `public/v2/`，让新版界面可以和经典版共存
- 通过 `/experience/v2` 与 `/experience/classic` 在新版 / 经典版之间切换
- 首批接入：首页、认证页、设置页、通知页和公开说明页
- `/profile`、`/matches`、`/admin` 仍保留经典版，避免把现有核心流程一起改乱

## 目录说明

- `frontend-redesign-pack-dark-mode-20260401/`
  - 已解压的独立前端原型目录
  - 包含多页面原型、样式、脚本、mock 数据、接口契约草案与设计说明
- `示例/`
  - 从原型中导出的预览截图，以及本次可运行新版预览的录屏
  - 其中 `屏幕录制 2026-04-02 225048.mp4` 用来演示这条分支当前已经接入的新版界面效果

## 本次交付范围

这次 PR 现在包含两部分：

1. **前端重设计交接物**
2. **基于当前单体 Express + EJS 项目的新版共存预览**

也就是说，它目前：
- 提供一套独立可审阅的前端原型
- 提供设计约束、页面结构和 mock 数据
- 为后续是否迁移到现有路由体系提供讨论基础
- 提供一版能直接在当前项目里运行的新版入口与页面壳层

它目前**不包含**：
- 对现有问卷页、匹配页和管理后台的完整新版迁移
- 对数据库结构或业务 API 的额外扩展
- 对整站视觉的最终定稿

## 本次新增改动

- 在 `app.js` 中加入 `ui_experience` cookie 切换机制
- 新增 `GET /experience/v2` 与 `GET /experience/classic`
- 新增新版模板目录 `views/v2/`
- 新增新版静态资源目录 `public/v2/`
- 首批把以下页面接成新版共存预览：
  - 首页
  - 登录 / 注册
  - 忘记密码 / 重置密码
  - 设置 / 修改密码 / 注销账号
  - 通知页
  - 隐私政策 / 使用说明
- 经典版导航和页脚加入“体验新版 / 返回经典版”入口
- 未接入新版的页面仍回退到经典版，保证现有流程可继续使用

## 建议评审重点

- 导航结构与移动端信息架构是否更清晰
- 用户菜单 / 设置 / 通知 / 匹配页的整体一致性
- 问卷页的布局、分组和可读性是否更好
- 深色模式视觉方向是否合理
- 在当前单体架构下，以“新版共存”方式逐步迁移的成本是否可控
