# UI / UX 约束

## 总体风格
- 校园感、轻量、清爽
- 蓝白主色调为主
- 卡片式内容区
- 不要做过度炫技动画
- 比起“花”，更重视信息清晰和可达性

## 交互约束
1. 关键导航不可依赖 hover
2. 所有下拉/菜单在手机触屏下必须稳定可用
3. 所有表单状态要有明确反馈
4. 空状态页面不能只有一句“暂无”，最好带解释或下一步引导
5. 高风险操作（注销账号）必须有明显风险提示

## 响应式约束
- Mobile first
- 断点至少考虑：
  - 手机：<= 480px
  - 平板：481px - 1024px
  - 桌面：> 1024px
- 手机端顶部导航不能挤爆
- 长昵称、长按钮文案不能导致布局崩坏

## 可访问性约束
- 交互控件必须有 focus 样式
- 菜单按钮要有 aria-expanded / aria-controls（如果是组件化实现）
- 颜色对比要足够
- 图标不能单独承担全部语义

## 内容约束
- 文案保持大学校园产品语气，不要太企业官网
- 不要堆砌空洞 slogan
- 页面标题、导航标题、按钮文案保持统一

## 技术约束
- 可以先用 mock data
- 不要求接入真实后端
- 输出结构要便于后续迁移到 Express/EJS 或独立前端
- 尽量减少内联 style
